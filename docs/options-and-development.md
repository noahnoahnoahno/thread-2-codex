# 영상 소개 프로그램 옵션 정리 및 디벨롭 방향

이 문서는 영상 자막에서 직접 확인되는 옵션과, 제품으로 발전시키기 위해 추가하면 좋은 옵션을 구분해 정리한다.

## 1. 영상에서 직접 소개된 옵션

### 입력 옵션

- 유튜브 URL 입력
- 쇼츠를 만들 롱폼 영상 지정
- 영상이 맞는지 확인하는 확인 창
- `분석 시작` 실행

### 처리 단계 옵션/상태

영상에서는 처리 흐름이 5단계로 표시된다.

1. 영상 다운로드
2. 오디오 추출
3. 음성 인식
4. AI 분석
5. 종합 생성

제품 관점에서는 이 단계들이 단순 설명이 아니라 작업 상태 UI가 되어야 한다. 각 단계별 진행률, 실패 사유, 재시도 버튼까지 확장 가능하다.

### AI 분석 옵션

- 음성과 내용을 분석
- 후킹되는 구간 자동 탐지
- 문맥을 파악해서 쇼츠 후보 선정
- 롱폼 하나당 최대 6개 후보 생성
- 후보별 영상 길이 표시
- 후보별 선정 근거 표시
- 후보별 영상 제목 자동 생성
- 추천 유튜브 해시태그 자동 생성

`최대 6개`는 영상에서 명확히 나온 기본값이다. 실제 제품에서는 사용자가 3개, 6개, 10개 등으로 선택하게 만들 수 있다.

### 후보 선택 옵션

- 생성된 후보 클립 중 하나 선택
- 선택한 클립의 상세 편집 패널 표시
- 선택된 한 개 클립만 생성
- 여러 후보를 각각 세부 조정한 뒤 일괄 생성

### 메타데이터 편집 옵션

- 영상 제목 표시 및 수정
- 추천 해시태그 표시
- 해시태그 수동 추가
- 후킹 선정 근거 확인

영상에서는 제목/해시태그 자동 생성과 태그 추가가 직접 언급된다. 제목 수정 가능 여부는 UI 맥락상 자연스럽지만, 자막에서 명확히 말한 것은 제목 자동 생성이다.

### 미리보기 옵션

- 쇼츠 화면 예시 미리보기
- 재생 미리보기
- 빨간색 구간 표시를 통한 선택 범위 미리보기
- 세이프존 테두리 표시
- 중앙 3분할 가이드라인 표시

### 레이아웃 옵션

- 레터박스 형식
  - 원본 비율 유지
  - 상단 제목 영역
  - 하단 채널명 영역
- 세로 크롭 형식
  - 9:16 화면을 꽉 채우는 방식

### 텍스트/브랜딩 옵션

- 상단 제목 표시
- 제목 위치 조정
- 제목 가로 중앙 정렬
- 하단 채널명 표시
- 채널명 입력
- 채널명 위치 조정
- 여러 글꼴 중 선택
- 텍스트 색상 변경

### 타임라인 편집 옵션

- 클립 전체 길이 조정
- 내용을 더 추가하기 위한 구간 확장
- 빼고 싶은 부분을 줄이기 위한 구간 축소
- 시작점 조정
- 끝점 조정
- 시작점/끝점 1초 단위 이동
- 저장

### 출력 옵션

- 선택된 한 개 클립 생성
- 여러 클립 생성
- MP4 다운로드
- 여러 개 클립 다운로드

## 2. MVP에 반드시 넣을 옵션

첫 버전에서 제품 가치가 살아나려면 아래 옵션은 빠지면 안 된다.

- 파일 업로드 또는 권한 있는 URL 입력
- 영상 확인 화면
- 5단계 처리 상태
- 후보 개수 설정: 3개/6개
- 후보별 제목, 길이, 선정 근거, 해시태그 표시
- 후보 선택
- 시작/끝 시간 조정
- 1초 단위 앞뒤 이동
- 레터박스/세로 크롭 선택
- 제목/채널명 텍스트 입력
- 제목/채널명 위치 조정
- 폰트 선택
- 색상 선택
- 세이프존 가이드
- 미리보기
- 단일 MP4 생성 및 다운로드

## 3. 제품화하면서 디벨롭할 옵션

### A. AI 후보 추천 고도화

- 후보 개수 직접 설정: 3, 6, 10, 15개
- 클립 길이 범위 설정: 15초, 30초, 60초, 직접 입력
- 콘텐츠 목적 선택
  - 조회수형
  - 정보 전달형
  - 논쟁/댓글 유도형
  - 구매/신청 전환형
- 톤 선택
  - 전문적
  - 자극적
  - 담백한 정보형
  - 예능형
- 후킹 유형 필터
  - 숫자/성과
  - 실수/실패
  - 고백/반전
  - 노하우
  - 질문형
  - 갈등/비교
- 중복 후보 제거 강도
- AI 추천 이유 상세/간단 보기
- 후보 점수 표시: hook score, clarity score, retention score

### B. 자막 편집 고도화

- 자동 자막 표시/숨김
- 자막 문장 단위 편집
- 금칙어/오타 자동 교정
- 말줄임, 반복어, 군더더기 제거
- 자막 줄 수 제한: 1줄/2줄/3줄
- 자막 위치 프리셋
- 자막 배경 박스/그림자/외곽선
- 단어별 하이라이트
- 키워드 자동 강조
- SRT/ASS 내보내기

### C. 화면 구성 고도화

- 레이아웃 프리셋 저장
- 브랜드 키트
  - 로고
  - 채널명
  - 기본 폰트
  - 브랜드 컬러
  - 제목 위치
- 얼굴/화자 자동 추적 크롭
- 수동 크롭 영역 조정
- 상단 제목 자동 줄바꿈
- YouTube Shorts UI 세이프존 프리셋
- TikTok/Reels 세이프존 프리셋
- 배경 블러/확대 배경
- 로고 워터마크

### D. 편집 생산성 옵션

- 모든 후보에 같은 스타일 적용
- 특정 클립 스타일 복사/붙여넣기
- 일괄 렌더링
- 렌더링 전 품질 체크
- 클립별 메모
- 후보 채택/보류/삭제 상태
- 프로젝트 자동 저장
- 이전 버전 복원

### E. 업로드/운영 옵션

- YouTube Shorts 제목/설명/태그 패키지 생성
- 썸네일 프레임 추천
- 업로드 예약 초안 생성
- YouTube 업로드 API 연동
- TikTok/Reels용 파일명/캡션 변환
- 플랫폼별 비율과 길이 제한 체크
- 게시 후 성과 기록
- 성과 좋은 클립 패턴 분석

### F. 협업/서비스화 옵션

- 로그인
- 프로젝트 목록
- 팀 워크스페이스
- 사용자별 저장 용량 제한
- 작업 큐 대기열
- 크레딧/사용량 과금
- 워터마크 무료 플랜
- 관리자 대시보드
- 신고/삭제 요청 대응

## 4. 구현 우선순위

### Phase 1: 핵심 제작 파이프라인

목표: 영상 하나를 넣고 쇼츠 후보를 만들어 MP4로 받는다.

- 파일 업로드
- 오디오 추출
- 음성 인식
- AI 후보 추천
- 후보 카드
- 시작/끝 조정
- 9:16 렌더링
- MP4 다운로드

### Phase 2: 영상에서 소개된 편집 경험 재현

목표: 영상 속 `아크 클리퍼`에 가까운 편집 UI를 만든다.

- 5단계 상태 UI
- 후보 최대 6개 생성
- 선정 근거/제목/해시태그
- 세이프존/3분할 가이드
- 레터박스/세로 크롭
- 제목/채널명 위치 조정
- 폰트/색상 선택
- 1초 단위 조정
- 일괄 생성

### Phase 3: 우리만의 차별화

목표: 단순 복제보다 더 좋은 제작 도구로 만든다.

- 후보 점수화
- 자막 자동 정리
- 브랜드 키트
- 얼굴/화자 추적 크롭
- 플랫폼별 프리셋
- 성과 기반 추천 개선
- 팀/프로젝트 관리

## 5. 주요 개발 판단

초기 MVP에서는 `완전 자동 생성`보다 `AI 추천 + 사람 검수 + 빠른 렌더링`을 우선한다. 쇼츠는 작은 오차도 결과물 품질을 크게 떨어뜨리기 때문에, 시작/끝 지점과 제목/자막 위치를 사람이 직접 조정할 수 있어야 한다.

또한 YouTube URL 다운로드는 권한과 약관 문제가 있을 수 있으므로, 초기 제품은 영상 파일 업로드를 기본 경로로 둔다. URL 입력은 사용자가 권한을 가진 영상이나 내부 테스트 환경부터 지원하는 방식이 안전하다.

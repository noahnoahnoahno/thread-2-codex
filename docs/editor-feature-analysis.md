# 편집 기능 분석: 타이틀 배치, 글자 입력, 레이아웃

이 문서는 영상에서 소개된 쇼츠 편집 기능을 실제 제품 기능 단위로 분해한 분석이다. 목표는 "AI가 뽑은 후보 클립을 사용자가 빠르게 쇼츠 형태로 다듬고 MP4로 렌더링하는 편집기"를 구현하는 것이다.

## 1. 영상에서 확인되는 편집 흐름

1. AI가 쇼츠 후보를 생성한다.
2. 사용자가 후보 영상 하나를 선택한다.
3. 하단 또는 상세 영역에서 편집 패널이 열린다.
4. 후보 선정 근거, 영상 제목, 추천 해시태그를 확인한다.
5. 쇼츠 미리보기 화면에서 레이아웃을 고른다.
6. 제목, 채널명, 폰트, 색상, 위치를 조정한다.
7. 시작점과 끝점을 조정한다.
8. 저장 후 선택 클립 또는 여러 클립을 MP4로 생성한다.

## 2. 타이틀 배치 기능

영상에서 명확히 소개된 타이틀 기능은 다음과 같다.

- 상단 제목 표시
- 제목 위치 이동
- 가로 중앙 정렬
- 여러 글꼴 중 선택
- 색상 변경
- 레터박스 레이아웃에서는 원본 영상 위쪽 여백에 제목 배치
- 세로 크롭 레이아웃에서는 화면 위에 제목 오버레이 가능

### 구현 기능 단위

#### Title Layer

타이틀은 렌더링 가능한 독립 레이어로 관리한다.

필요 속성:

- text: 표시할 제목
- visible: 표시 여부
- x: 기준 좌표
- y: 기준 좌표
- anchor: left | center | right
- font_family
- font_size
- font_weight
- color
- stroke_color
- stroke_width
- background_enabled
- background_color
- max_width
- line_count_limit
- align: left | center | right

### UI 컨트롤

- 제목 입력 필드
- 위치 드래그
- `가로 중앙 정렬` 버튼
- 폰트 선택 드롭다운
- 기본 타이틀 폰트: `NEXON Lv1 Gothic`
- 글자 크기 슬라이더
- 글자 색상 선택
- 외곽선/그림자 토글
- 줄바꿈 미리보기
- 제목 표시/숨김 토글

### 개발 시 주의점

- 쇼츠 UI 상단 영역과 겹치지 않도록 세이프존을 기본 표시한다.
- 긴 제목은 자동 줄바꿈이 필요하다.
- 모바일 화면에서도 제목이 화면 밖으로 나가면 안 된다.
- 미리보기 좌표와 FFmpeg 렌더링 좌표가 일치해야 한다.
- 폰트 파일이 서버 렌더링 환경에도 있어야 한다.

## 3. 채널명/하단 글자 기능

영상에서는 하단 채널명을 입력하고 위치를 이동할 수 있다고 설명한다.

### 구현 기능 단위

#### Channel Label Layer

필요 속성:

- text: 채널명 또는 브랜딩 문구
- visible
- x
- y
- anchor
- font_family
- font_size
- color
- opacity
- background_enabled
- background_color

### UI 컨트롤

- 채널명 입력 필드
- 위치 드래그
- 하단 중앙 정렬 버튼
- 폰트 선택
- 글자 색상 선택
- 표시/숨김 토글

### 개발 시 주의점

- YouTube Shorts 하단 버튼/캡션 UI와 겹치지 않는 위치 프리셋이 필요하다.
- 레터박스 레이아웃에서는 하단 여백 안에 넣는 방식이 안정적이다.
- 세로 크롭에서는 영상 위 오버레이가 되므로 배경 대비가 낮으면 가독성이 떨어진다.

## 4. 자막 기능

영상에서는 "제목이랑 자막까지 달아준다"고 설명한다. 자세한 자막 스타일 옵션은 많이 소개되지 않았지만, 제품에는 반드시 들어가야 하는 핵심 기능이다.

### MVP 자막 기능

- 전사 결과 기반 자동 자막 생성
- 자막 표시/숨김
- 자막 위치 선택: 하단, 중앙 하단
- 자막 색상
- 자막 외곽선
- 자막 배경 박스
- 줄 수 제한

### 디벨롭 기능

- 단어별 하이라이트
- 핵심 키워드 자동 강조
- 자막 문장 단위 편집
- 반복어/군더더기 제거
- 오타 교정
- SRT/ASS 내보내기

## 5. 레이아웃 기능 분석

영상에서 소개된 레이아웃은 두 가지다.

### 레터박스

원본 비율을 유지한 채 9:16 캔버스에 올리고, 위아래 여백을 사용한다.

포함 요소:

- 상단 제목 영역
- 중앙 원본 영상
- 하단 채널명 영역

장점:

- 원본 구도가 잘리지 않는다.
- 제목과 채널명을 안정적으로 배치하기 쉽다.
- 인터뷰/강의/팟캐스트 콘텐츠에 적합하다.

단점:

- 화면 몰입감은 세로 크롭보다 낮을 수 있다.

### 세로 크롭

원본 영상을 9:16 화면에 꽉 차게 자른다.

포함 요소:

- 전체 화면 영상
- 제목/자막/채널명 오버레이

장점:

- 쇼츠 피드에서 몰입감이 높다.
- 인물 중심 콘텐츠에 적합하다.

단점:

- 중요한 인물이 화면 밖으로 잘릴 수 있다.
- 자동 크롭 품질이 중요하다.

## 6. 가이드 기능

영상에서는 세이프존 테두리와 중앙 3분할 가이드라인이 보인다.

### 세이프존

목적:

- 제목, 자막, 채널명이 플랫폼 UI에 가려지지 않게 한다.
- 렌더링 전 배치 오류를 줄인다.

구현:

- 미리보기 위에 반투명 가이드 오버레이
- YouTube Shorts 기본 세이프존
- 추후 TikTok/Reels 프리셋 추가

### 중앙 3분할 가이드

목적:

- 제목이나 인물 중심을 맞추기 쉽게 한다.
- 수동 크롭/텍스트 배치에 도움을 준다.

구현:

- 3x3 그리드 표시/숨김
- 중앙선 강조
- 드래그 시 스냅 옵션

## 7. 타임라인/구간 조정 기능

영상에서 확인되는 기능:

- 클립 길이 조정
- 내용을 더 추가하기 위한 구간 확장
- 빼고 싶은 부분을 제거하기 위한 구간 축소
- 시작점 조정
- 끝점 조정
- 1초 전/후 이동
- 빨간색 선택 구간 미리보기

### 구현 기능 단위

- start_sec
- end_sec
- duration_sec
- timeline selection range
- nudge start -1s/+1s
- nudge end -1s/+1s
- preview selected range
- save clip edit

### 디벨롭 기능

- 자막 문장 경계에 맞춰 스냅
- 무음 구간 자동 제거
- 클립 길이 제한 경고
- 앞뒤 여유 1초 자동 추가
- 컷 전후 미리보기

## 8. 미리보기와 렌더링 일치 전략

편집기의 가장 중요한 품질 기준은 "브라우저에서 본 결과와 다운로드한 MP4가 일치하는 것"이다.

### 미리보기

- 9:16 고정 캔버스
- 레이어 기반 렌더링
- 제목/채널명/자막/가이드 오버레이
- 드래그 위치 조정
- 재생 미리보기

### 서버 렌더링

- 같은 좌표계를 FFmpeg 필터로 변환
- 폰트 파일 경로 고정
- 텍스트 줄바꿈 로직 공유
- 1080x1920 기준 좌표 저장
- 모든 위치 값은 비율 또는 기준 해상도 좌표로 저장

권장 방식:

- 편집 좌표는 1080x1920 기준으로 저장한다.
- 프론트엔드 미리보기에서는 현재 화면 크기에 맞춰 비례 축소한다.
- FFmpeg 렌더링에서는 저장된 1080x1920 좌표를 그대로 사용한다.

## 10. 프로젝트 포함 범위

### MVP 포함

- 제목 입력 및 자동 제목 반영
- 제목 위치 드래그
- 제목 가로 중앙 정렬
- 채널명 입력
- 채널명 위치 드래그
- 폰트 선택
- 텍스트 색상 선택
- 레터박스/세로 크롭 선택
- 세이프존 표시
- 3분할 가이드 표시
- 시작/끝 시간 조정
- 1초 단위 이동
- 선택 클립 MP4 렌더링

### MVP 이후

- 텍스트 외곽선/그림자/배경 박스
- 브랜드 키트 저장
- 자막 문장별 편집
- 키워드 하이라이트
- 얼굴/화자 추적 크롭
- 플랫폼별 세이프존 프리셋
- 모든 후보에 스타일 일괄 적용
